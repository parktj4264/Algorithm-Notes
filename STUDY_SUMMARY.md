### 🧠 실전에서 무기를 꺼내는 알고리즘 (머릿속 의사결정 트리)

실전(면접장)에 가면 문제가 "단조 스택으로 푸세요"라고 절대 안 알려줘. 대신 문제의 **'제약 조건(Data Size)'**과 **'요구 사항'**이 네 머릿속 트리 모델의 분기점(Split Node)이 될 거야.

예를 들어볼까?

1. **데이터 길이가 $N = 10^5$ (10만 개)이다?**
    * *너의 뇌:* "아, 무지성 $O(N^2)$ 하면 100억 번이라 무조건 타임아웃(TLE) 난다. $O(N)$이나 $O(N \log N)$으로 찢어야 해."
    * *후보군 압축:* 해시(Dict/Set), 투 포인터, 슬라이딩 윈도우, 스택/큐, 정렬.


2. **근데 문제에서 '연속된' 구간의 최댓값을 구하래.**
    * *너의 뇌:* "연속된 구간? 포인터가 뒤로 갈 필요 없네." -> **슬라이딩 윈도우 채택!**


3. **문제에서 '최단 거리'나 '최소 횟수'를 구하래.**
    * *너의 뇌:* "공간 탐색인데 최단거리? 깊게 파는 DFS 말고 넓게 퍼지는 BFS 써야겠네." -> **큐(Queue) 채택!**


4. **문제에서 '고정된 크기의 배열'이 주어지고, 데이터의 삽입/삭제(FIFO)가 무한히 반복된다면?**
* *너의 뇌:* "선형 큐를 쓰면 삭제된 앞쪽 공간을 버리거나 당겨오는($O(N)$) 비용이 발생하네. 한정된 메모리 공간을 순환시키며 재활용해야겠다." -> **원형 큐 (Modulo `%` 연산) 채택!**


5. **문제에서 '2차원 격자(Grid)'나 '네트워크/노드'가 주어지고, '최단 거리/최소 이동 횟수' 또는 '연결된 덩어리(Connected Component)'를 구하래.**
* *너의 뇌:* "깊게 파고들면 사이클에 빠지거나 최단 거리를 보장할 수 없으니, 시작점에서 가까운 노드부터 층별(Level)로 균일하게 탐색해야겠다." -> **BFS (큐) 채택!**


6. **문제에서 '모든 경우의 수'를 다 탐색하거나, '미로의 끝까지' 일단 가봐야 한대.**
* *너의 뇌:* "일단 바닥을 찍을 때까지 깊게 파고들고, 막히면 방금 전 갈림길로 돌아와야겠네." -> **DFS (스택/재귀) 채택!**

# PART 0. 컴퓨터 메모리(RAM)의 이해

알고리즘과 자료구조의 본질은 결국 **"한정된 RAM 공간을 얼마나 찰지게 쓰고(공간 복잡도), 얼마나 빨리 찾느냐(시간 복잡도)"**의 싸움이다. 메모리 주소(`0x...`)를 외울 필요는 없지만, 데이터가 어떻게 배치되는지 '논리적 형태'는 반드시 머리에 박아둬야 한다.

## 🧱 1. 연속적 할당 (Contiguous Memory) - 배열, 리스트

* **특징:** 데이터가 RAM 공간에 다닥다닥 붙어있다.
* **장단점:** 인덱스로 찌르면 $O(1)$로 빛의 속도 접근이 가능하지만, 중간에 데이터를 끼워 넣거나 빼려면 뒤에 있는 놈들을 다 밀고 당겨야 해서 $O(N)$의 노가다가 발생한다.
* **시각화:**

```text
 RAM: [ 빈공간 ] [ 데이터1 | 데이터2 | 데이터3 ] [ 빈공간 ] 
                  ⬆️ 인덱스 [1] 찌르면 바로 "데이터2" 나옴!

```

* **💻 파이썬 실전 코드 (배열의 딜레마):**

```python
arr = [10, 20, 30]

# 🟢 장점 (빛의 속도 탐색): 시작 주소에서 딱 1칸만 이동하면 됨
print(arr[1]) # 20 (시간 복잡도 O(1))

# 🔴 단점 (메모리 대공사): 0번 인덱스에 5를 넣으려면 10, 20, 30을 다 뒤로 밀어야 함
arr.insert(0, 5) # 상태: [5, 10, 20, 30] (시간 복잡도 O(N))
arr.pop(0)       # 뺄 때도 남은 애들 다 앞으로 당겨야 함 (시간 복잡도 O(N))

```

## 🔗 2. 분산 할당 (Scattered Memory) - 그래프, 트리, 연결 리스트

* **특징:** 데이터가 RAM 여기저기 파편화되어 흩어져 있고, 서로의 '위치(참조/Pointer)'를 화살표로 기억하고 있다.
* **장단점:** 새로운 데이터를 허공에 띡 만들고 화살표만 쓱 연결하면 되니까 삽입/삭제가 매우 유연하다($O(1)$). 하지만 3번째 데이터를 찾으려면 1번부터 화살표를 타고 넘어가야 해서 탐색이 느리다($O(N)$).
* **시각화:**

```text
 RAM: [ 데이터1 ] ➔ ➔ ➔ (허공) ➔ ➔ ➔ [ 데이터2 ]
         ⬇️                               ⬇️
       (허공)                            (허공)
         ⬇️                               ⬇️
      [ 데이터3 ] ⬅️ ⬅️ ⬅️ (허공) ⬅️ ⬅️ ⬅️ [ 데이터4 ]
```

* **💻 파이썬 실전 코드 (참조로 연결하기):**

```python
# 노드라는 '그릇'을 메모리 허공에 둥둥 띄우는 클래스
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None  # 화살표(포인터) 역할

# 1. 메모리 여기저기에 흩어진 데이터 생성
node1 = Node("데이터1")
node2 = Node("데이터2")

# 2. 🟢 장점 (유연한 연결): 노드들을 메모리상에 밀고 당길 필요 없이 화살표만 이어줌!
node1.next = node2 # 데이터1이 데이터2를 가리킴 (시간 복잡도 O(1))

# 3. 🔴 단점 (느린 탐색): 데이터2를 찾으려면 무조건 데이터1을 거쳐서 가야 함
print(node1.next.data) # "데이터2" (순차적으로 화살표를 타야 함)
```

# PART 1. 자료구조 (Data Structures)
데이터를 메모리에 담고 접근하는 '그릇'의 형태. 적절한 자료구조의 선택이 시간 복잡도 최적화의 첫걸음이다.

데이터를 메모리에 담고 접근하는 '그릇'의 형태. 적절한 자료구조의 선택이 시간 복잡도 최적화의 첫걸음이다.

### 1. 리스트와 문자열 (List & String)

* **개념:** 순서(Index)가 있는 1차원 데이터 배열. 파이썬의 문자열은 불변(Immutable) 객체.
* **시간 복잡도:** 인덱스 접근 $O(1)$. 탐색/삽입/삭제 $O(N)$.
* **💻 파이썬 실전 문법:**
```python
lst.append(x)      # 맨 뒤 삽입 O(1)
lst.pop()          # 맨 뒤 삭제 O(1)
lst.pop(0)         # 🚨 맨 앞 삭제 O(N) -> 절대 쓰지 말 것!
text[::-1]         # 문자열/리스트 뒤집기 O(N)
```


* **특수문자 시각화:**

```text
 Index:   0     1     2     3
       +-----+-----+-----+-----+
 Data: | 'a' | 'b' | 'c' | 'd' |
       +-----+-----+-----+-----+
```

### 2. 해시 테이블 - 딕셔너리 (Dictionary)

* **개념:** Key-Value 쌍으로 이루어진 룩업 테이블(Lookup Table). 데이터의 고유한 매핑(Mapping)을 저장.
* **시간 복잡도:** 탐색(`in`), 삽입, 삭제 모두 완벽한 **$O(1)$**.
* **💻 파이썬 실전 문법:**
```python
my_dict[key] = val        # 삽입 및 갱신 O(1)
if key in my_dict: ...    # 🚨 값(val)이 아닌 키(key) 존재 여부 탐색 O(1)
val = my_dict.get(key, 0) # 키가 없으면 에러 대신 기본값(0) 반환 (개꿀팁!)
```


* **특수문자 시각화:**

```text
 [Key]       [Hash Engine]       [Value]
"apple"  ➔  ( %$^#@! )  ➔   Idx 2 : 500원
"banana" ➔  ( *&^%$# )  ➔   Idx 5 : 300원
```

### 3. 해시 테이블 - 집합 (Set)

* **개념:** Value 없이 Key만 저장하여 '중복을 제거'한 자료구조. 데이터의 "존재 여부"를 빠르게 확인할 때 사용.
* **시간 복잡도:** 탐색(`in` 연산) **$O(1)$**.
* **💻 파이썬 실전 문법:**
```python
my_set = set([1, 2, 2, 3]) # 리스트를 O(N)만에 중복 제거된 셋으로 변환 -> {1, 2, 3}
my_set.add(4)              # 데이터 추가 O(1)
my_set.remove(4)           # 데이터 삭제 O(1)
if target in my_set: ...   # 🚨 리스트의 in은 O(N)이지만 셋의 in은 O(1)!
```


* **특수문자 시각화:**

```text
 Input: [1, 2, 2, 3, 3, 3]
            ⬇️ (Set 통과)
 Output: ( 1, 2, 3 )  <-- 중복 증발!
```

### 4. 카운터 (collections.Counter)

* **개념:** 해시 테이블 기반으로 데이터의 등장 빈도(Frequency)를 자동 집계. (통계의 빈도표)
* **시간 복잡도:** 집계 시 **$O(N)$**.
* **💻 파이썬 실전 문법:**
```python
from collections import Counter
counts = Counter(["A", "A", "B"]) # Counter({'A': 2, 'B': 1})
counts.most_common(1)             # 빈도수 1등 뽑기 -> [('A', 2)] O(N log K)
```


* **특수문자 시각화:**

```text
 [ 'A', 'B', 'A', 'C', 'A', 'B' ]
            ⬇️ (Counter)
 { 'A': 3,  'B': 2,  'C': 1 }
```

### 5. 힙 (Heap / Priority Queue)

* **개념:** 항상 '최솟값(또는 최댓값)'이 맨 위에 오도록 반쯤 정렬된 상태를 유지하는 이진 트리. Top-K 추출에 최적.
* **시간 복잡도:** 삽입/삭제 **$O(\log N)$**.
* **💻 파이썬 실전 문법:**
```python
import heapq
hq = []
heapq.heappush(hq, 5)    # 힙에 데이터 넣기 O(log N)
min_val = heapq.heappop(hq) # 가장 작은 값 빼기 O(log N)
# 🚨 파이썬은 최소 힙(Min-Heap)만 지원! 최대 힙은 넣을 때 -1을 곱해서 넣어야 함.
```


* **특수문자 시각화 (최소 힙):**

```text
        [ 1 ]    <-- 최솟값 (항상 O(1)로 확인 가능: hq[0])
       /     \
    [ 3 ]   [ 5 ]
    /   \
 [ 7 ] [ 9 ]
```

### 6. 스택 (Stack)

* **개념:** 나중에 들어온 데이터가 먼저 나가는 **LIFO**. 과거 이력 기억, DFS 백트래킹에 사용. 파이썬은 기본 리스트를 씀.
* **시간 복잡도:** 삽입/삭제 모두 **$O(1)$**.
* **💻 파이썬 실전 문법:**
```python
stack = []
stack.append(x)      # Push (맨 뒤에 넣기)
top_item = stack.pop() # Pop (맨 뒤에서 빼기)
peek_item = stack[-1]  # Top (빼지 않고 맨 위 값만 확인)
if stack: ...        # 스택이 비어있지 않은지 확인 (len(stack) > 0 보다 Pythonic!)
```


* **특수문자 시각화:**

```text
       |       |  ⬆️ Pop (3 나감)
       | [ 3 ] |  
       | [ 2 ] |  ⬇️ Push
       | [ 1 ] |  
       +-------+
```

### 7. 단조 스택 (Monotonic Stack)

* **개념:** 스택 내부 원소가 오름차순/내림차순을 엄격히 유지하도록 통제. "나보다 큰 다음 값" 찾기에 사용.
* **시간 복잡도:** **$O(N)$** (분할 상환 분석)
* **💻 파이썬 실전 문법:**
```python
stack = []
for i, current in enumerate(nums):
    # 현재 값이 스택 맨 위(과거) 값보다 크면 조건 달성 -> 청산(Pop)
    while stack and current > stack[-1]:
        past_val = stack.pop() 
    stack.append(current) # 새로운 값 대기실 입장
```


* **특수문자 시각화:**

```text
 현재 스택: | [ 10 ] |  <-- 대기 중인 과거 데이터들
           | [ 20 ] |  
           +--------+
 🚨 새로운 데이터 [ 30 ] 등장! (10, 20보다 큼)
 💥 10, 20은 조건 달성으로 Pop(청산)되고 30이 들어감!
```

### 8. 큐와 데크 (Queue & Deque)

* **개념:** 먼저 들어온 데이터가 먼저 나가는 **FIFO**. 공평한 순서 처리, BFS 탐색에 사용.
* **시간 복잡도:** 양 끝 삽입/삭제 모두 **$O(1)$**.
* **💻 파이썬 실전 문법:**
```python
from collections import deque
q = deque([1, 2, 3])
q.append(4)          # 맨 뒤 삽입 (enQueue)
first = q.popleft()  # 🚨 맨 앞 추출 (deQueue) -> 절대 pop(0) 쓰지 말 것!
```


* **특수문자 시각화:**

```text
            +---+---+---+
 IN (Push) ➔ | 3 | 2 | 1 | ➔ OUT (popleft, 1 나감)
            +---+---+---+
```

### 9. 원형 큐 (Circular Queue / Ring Buffer)

* **개념:** 한정된 배열을 둥글게 이어 붙여 삭제된 앞쪽 공간을 재활용. 상태 관리는 `size` 변수 하나로 통제!
* **시간 복잡도:** 삽입/삭제 모두 **$O(1)$**.
* **💻 파이썬 실전 문법 (모듈로 `%` 치트키):**
```python
self.q = [None] * k
self.rear = (self.rear + 1) % k  # 끝에 닿으면 0으로 회전
self.q[self.rear] = value        # 무자비한 덮어쓰기 (None 탐색 X)
```


* **특수문자 시각화:**

```text
       [0] ➔ ➔ ➔ [1]
        ⬆️         ⬇️
 [3] ⬅️ ⬅️ ⬅️ [2]
 (Rear가 3번에 꽉 차면, 다음 삽입은 다시 0번으로 덮어쓰기 됨!)
```

### 10. 트리 (Tree)

* **개념:** 부모-자식 계층적 구조. 사이클(순환)이 절대 없다.
* **💻 파이썬 실전 문법 (클래스 구조):**
```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left   # 왼쪽 자식 노드 연결
        self.right = right # 오른쪽 자식 노드 연결
```


* **특수문자 시각화:**

```text
       [Root]          <-- Level 0
      /      \
   [A]        [B]      <-- Level 1
   /  \         \
 [C]  [D]       [E]    <-- Level 2 (Leaves)
```

### 11. 그래프 (Graph)

* **개념:** 정점(Node)과 간선(Edge)으로 연결된 네트워크. 사이클(무한 루프)이 존재할 수 있어 '방문 처리' 필수.
* **💻 파이썬 실전 문법 (인접 리스트):**
```python
from collections import defaultdict
graph = defaultdict(list)
graph['A'].append('B') # A에서 B로 가는 길 생성
graph['A'].append('C') # A에서 C로 가는 길 생성
```


* **특수문자 시각화:**

```text
  (A) ㅡㅡㅡ (B)
   |  \       |
   |    \     |
  (C) ㅡㅡㅡ (D)   <-- A-B-D-C-A 로 무한히 도는 사이클 존재!
```

# PART 2. 알고리즘 (Algorithms)

무지성 이중/삼중 반복문($O(N^2), O(N^3)$)을 단일 탐색($O(N)$)으로 낮추는 핵심 논리.

### 1. 정렬 (Sorting)

* **개념:** 무작위 데이터를 오름차순/내림차순으로 재배열하여 데이터에 '방향성'을 부여하는 기법.
* **활용 목적:** 투 포인터나 이진 탐색의 전제 조건. 중복 데이터를 인접하게 모아 필터링을 쉽게 만들 때 사용.
* **시간 복잡도:** **$O(N \log N)$** (파이썬 내장 Timsort 기준).
* **💻 파이썬 실전 문법 (Custom 정렬 치트키):**
```python
nums.sort()                  # 원본 리스트 제자리 정렬 O(N log N)
new_nums = sorted(nums)      # 원본 유지, 새로운 정렬 리스트 반환

# 🚨 람다(lambda)를 이용한 다중 조건 정렬 (코테 단골!)
# 1순위: 길이 짧은 순, 2순위: 사전 순
words.sort(key=lambda x: (len(x), x)) 
```


* **특수문자 시각화:**

```text
 [ 3,  1,  2,  5,  4 ]
          ⬇️ (Sorting)
 [ 1,  2,  3,  4,  5 ] ➔ 방향성이 생겨서 탐색이 쉬워짐!
```

### 2. 투 포인터 (Two Pointers)

* **개념:** (주로 정렬된) 1차원 배열에서 두 개의 포인터(`left`, `right`)를 양 끝에 배치하고, 타겟 조건에 맞춰 중앙을 향해 좁혀 들어오는 탐색 기법.
* **활용 목적:** 두 원소의 합/차를 구하거나, 팰린드롬(회문)을 검사할 때 이중 for문($O(N^2)$)을 $O(N)$으로 대체.
* **시간 복잡도:** 포인터들이 교차할 때까지만 1회 순회하므로 **$O(N)$**.
* **💻 파이썬 실전 문법:**
```python
left, right = 0, len(nums) - 1
while left < right:            # 포인터가 교차하면 종료
    total = nums[left] + nums[right]
    if total < target:
        left += 1              # 합이 모자라면 왼쪽을 올려서 값을 키움 (정렬 됨 가정)
    elif total > target:
        right -= 1             # 합이 넘치면 오른쪽을 내려서 값을 줄임
    else:
        return [left, right]   # 정답 발견!
```


* **특수문자 시각화 (양 끝에서 조여오기):**

```text
  [ 1,  2,  3,  4,  5,  8,  9 ]
    ⬆️                      ⬆️
   Left (커져라!)         Right (작아져라!)
```

### 3. 슬라이딩 윈도우 (Sliding Window)

* **개념:** 배열 위에서 일정 구간(Window)을 설정하고, 이 창틀을 한쪽 방향(우측)으로 밀고 가며 조건을 검사하는 기법. (시계열 데이터의 이동평균 탐색과 동일).
* **활용 목적:** **'연속된'** 부분 배열이나 부분 문자열을 다룰 때 사용. 창문이 절대 뒤로 후진하지 않으므로 무지성 탐색을 방지함.
* **시간 복잡도:** 우측 끝(`end`)과 좌측 끝(`start`) 포인터가 각각 최대 $N$번 이동하므로 **$O(N)$**.
* **💻 파이썬 실전 문법 (가변 윈도우 템플릿):**
```python
start = 0
max_len = 0
for end in range(len(nums)):
    # 1. 윈도우에 새로운 값 추가 및 상태 업데이트

    # 2. 조건 위반 시, 조건을 다시 만족할 때까지 윈도우 축소 (start 전진)
    while condition_is_invalid():
        # 상태 업데이트 (start가 가리키는 값 제거)
        start += 1

    # 3. 유효한 윈도우의 최대/최소 길이 갱신
    max_len = max(max_len, end - start + 1)
```


* **특수문자 시각화 (창문 밀기):**

```text
 [ 1,  2,  3 ]  4,  5,  6      <-- Size 3 윈도우
   ➔   1 [ 2,  3,  4 ]  5,  6  <-- 한 칸 이동 (1 빠지고 4 들어옴)
```

### 4. 깊이 우선 탐색 (DFS: Depth-First Search)

* **개념:** 갈림길이 나오면 **바닥(Leaf)을 찍을 때까지 직진**하고, 막히면 가장 최근 갈림길로 되돌아가(Backtracking) 다른 길을 파는 탐색 기법.
* **활용 목적:** 모든 가능한 경우의 수(조합, 순열)를 다 탐색하거나, 끝까지 가봐야 결과를 알 수 있는 퍼즐/미로 찾기에 사용.
* **시간 복잡도:** 노드 $V$개, 간선 $E$개일 때 **$O(V + E)$**.
* **💻 파이썬 실전 문법 (재귀 함수 템플릿):**
```python
visited = set()

def dfs(node):
    # 1. 종료 조건 (Base Case) 또는 방문 처리
    if node in visited:
        return
    visited.add(node)

    # 2. 현재 노드에서 할 일 처리 (정답 기록 등)

    # 3. 인접한 다음 노드들로 더 깊게 파고들기
    for next_node in graph[node]:
        dfs(next_node)
```


* **특수문자 시각화 (수직 드릴 & 백트래킹):**

```text
 [1] 시작 ➔ [2] 직진 ➔ [3] 바닥 찍음 (막힘!)
                         ⬇️ (백트래킹)
            [5] 바닥 ⬅️ [4] 아까 안 간 다른 길
```

### 5. 너비 우선 탐색 (BFS: Breadth-First Search)

* **개념:** 시작점에서 돌멩이를 던진 것처럼, **가까운 노드(거리 1)부터 우선적으로 수평 확장**해 나가는 탐색 기법.
* **활용 목적:** 가중치가 없는 공간에서 **'최단 거리/최소 이동 횟수'**를 구하거나, 특정 조건을 만족하는 **'연결된 네트워크 그룹(섬)'**을 찾을 때 1순위.
* **시간 복잡도:** **$O(V + E)$** (2차원 격자는 **$O(M \times N)$**).
* **💻 파이썬 실전 문법 (데크 + 4방향 템플릿):**
```python
from collections import deque
directions = [(-1, 0), (1, 0), (0, -1), (0, 1)] # 상하좌우

def bfs(start_r, start_c):
    q = deque([(start_r, start_c)])
    grid[start_r][start_c] = 'VISITED' # 🚨 큐에 넣자마자 방문 처리 필수!

    while q:
        curr_r, curr_c = q.popleft() # 가장 가까운 파동부터 꺼냄

        for dr, dc in directions:
            nr, nc = curr_r + dr, curr_c + dc
            if 0 <= nr < m and 0 <= nc < n: # 경계선 체크
                if grid[nr][nc] == 'TARGET': 
                    q.append((nr, nc))
                    grid[nr][nc] = 'VISITED' # 파동 닿은 곳 즉시 방문 처리
```


* **특수문자 시각화 (수평 파동):**

```text
       (( [시작] ))       <-- 거리 0 (먼저 큐에 들어감)
   ((( 주변 1촌들 )))     <-- 거리 1
 (((( 주변 2촌들 ))))     <-- 거리 2 (나중에 큐에 들어감)
```



### 6. 이진 탐색 (Binary Search)

* **개념:** **"정렬된"** 배열에서 탐색 범위를 반씩(1/2) 쪼개 나가며 타겟을 찾는 기법. 통계학에서 누적분포함수(CDF)를 이용해 특정 분위수(Quantile)나 중앙값을 찾을 때 범위를 좁혀가는 과정과 완벽히 동일.
* **활용 목적:** 무지성 선형 탐색($O(N)$)을 기적의 속도로 찢어버릴 때 사용. 배열 탐색뿐만 아니라 "조건을 만족하는 최적값(Parametric Search)"을 찾을 때도 쓰임.
* **시간 복잡도:** 매 단계마다 표본 공간이 반갈죽 되므로 **$O(\log N)$**.
* **💻 파이썬 실전 문법 (bisect 치트키 & 템플릿):**

```python
import bisect
# 1. 내장 모듈 사용 (가장 Pythonic!)
idx = bisect.bisect_left(nums, target)  # target이 들어갈 왼쪽 인덱스 반환 O(log N)

# 2. 정석 while문 템플릿
left, right = 0, len(nums) - 1
while left <= right:
    mid = left + (right - left) // 2  # 오버플로우 방지 및 중앙값 찾기
    if nums[mid] == target:
        return mid
    elif nums[mid] < target:
        left = mid + 1                # 타겟이 더 크면 오른쪽 절반 탐색
    else:
        right = mid - 1               # 타겟이 더 작으면 왼쪽 절반 탐색

```

* **특수문자 시각화 (반갈죽):**

```text
 [ 1,  3,  5,  7,  9,  11,  13 ]   <-- Target: 11
           ⬆️ (Mid=7, 11보다 작네? 왼쪽 다 버려!)
                 [ 9,  11,  13 ]
                       ⬆️ (Mid=11, 찾았다!)

```

### 7. 동적 계획법 (DP: Dynamic Programming)

* **개념:** "과거의 정답을 기억해 두었다가 재활용한다." 통계학의 점화식(Recurrence Relation)이나 마르코프 체인(Markov Chain)처럼, 현재의 상태가 이전 상태들의 조합으로 이루어질 때 **메모리를 제물로 바쳐 중복 연산을 파괴**하는 최적화 기법.
* **활용 목적:** 피보나치 수열처럼 재귀 트리에서 동일한 하위 문제(Sub-problem)가 미친 듯이 반복될 때, 결과를 배열(Memoization/Tabulation)에 저장하여 꺼내 씀.
* **시간 복잡도:** 무지성 재귀 **$O(2^N)$** ➔ DP 적용 시 **$O(N)$**.
* **💻 파이썬 실전 문법 (Top-down 캐싱 & Bottom-up):**

```python
# 1. Pythonic 치트키: LRU Cache (재귀 함수 위에 달아주면 알아서 캐싱됨!)
from functools import lru_cache

@lru_cache(maxsize=None)
def fibo(n):
    if n <= 1: return n
    return fibo(n-1) + fibo(n-2)

# 2. 정석 Bottom-up (1차원 DP 테이블)
dp = [0] * (n + 1)
dp[1] = 1
for i in range(2, n + 1):
    dp[i] = dp[i-1] + dp[i-2]  # 점화식 그대로 구현

```

* **특수문자 시각화 (메모이제이션의 기적):**

```text
        [ F(4) ]
       /        \
   [ F(3) ]    [ F(2) ]  <-- 우측 F(2)는 계산 안 함! 
   /      \                  왼쪽에서 F(3) 구할 때 메모장(dp 배열)에
[ F(2) ] [ F(1) ]            적어둔 값 그냥 O(1)로 꺼내 옴. 

```

### 8. 순열과 조합 (Combinatorics - `itertools`)

* **개념:** 우리가 어제 뼈 빠지게 DFS 백트래킹으로 구현했던 순열($nPr$)과 조합($nCr$)을 C언어 기반의 초고속 내장 라이브러리로 뽑아내는 파이썬의 축복.
* **활용 목적:** 코테에서 "모든 경우의 수를 다 구하시오" 류의 완전 탐색(Brute-force) 문제가 나왔을 때, 백트래킹을 직접 짜지 않아도 될 때 사용. (단, 문제에서 백트래킹 자체를 묻거나 제약 조건이 복잡하면 직접 짜야 함).
* **💻 파이썬 실전 문법:**

```python
from itertools import permutations, combinations

nums = [1, 2, 3]
# 순열 (순서 O)
list(permutations(nums, 2))  # [(1, 2), (1, 3), (2, 1), (2, 3), (3, 1), (3, 2)]

# 조합 (순서 X)
list(combinations(nums, 2))  # [(1, 2), (1, 3), (2, 3)]

```
