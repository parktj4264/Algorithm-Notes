# 2단계: 문제 및 가이드 제시 (The 'For-loop' Perspective)

### 🚀 문제 424: Longest Repeating Character Replacement
### 🔗 **링크:** [https://leetcode.com/problems/longest-repeating-character-replacement/](https://leetcode.com/problems/longest-repeating-character-replacement/)

## **📜 문제 설명 (Mission Briefing):**
대문자로만 이루어진 문자열 `s`와 정수 `k`가 주어져. 형은 이 문자열에서 최대 **`k`개의 문자를 다른 대문자로 바꿀 수 있는 치트키**를 가지고 있어. 이 치트키를 써서 만들 수 있는 **'모두 같은 알파벳으로 이루어진 가장 긴 연속 부분 문자열'**의 길이를 구해야 해.

* **예시 1:** `s = "ABAB", k = 2`
    * 'A' 2개를 'B'로 바꾸면 ➔ "BBBB" (길이 4)
    * 정답: **4**
* **예시 2:** `s = "AABABBA", k = 1`
    * 가운데 있는 'B' 하나를 'A'로 바꾸면 ➔ "AAAABBA"
    * 정답: **4** (연속된 A가 4개)

## **🎯 오늘 부술 for문 (Big-O 관점):**
문자열 최대 길이가 **$10^5$ (10만 개)**야. 
이걸 무지성 이중 for문으로 "여기서부터 저기까지 윈도우 잡고, 알파벳 몇 개인지 세어보자~" 하면 $O(N^2)$이 돼서 연산량이 $10^{10}$ (100억 번), 즉 **타임아웃(TLE)으로 서버 터진다.**

이걸 **$O(N)$**으로 찢어버리려면, 윈도우의 시작점(`left`)과 끝점(`right`)을 잡고 한쪽 방향으로만 스윽 밀고 나가는 **슬라이딩 윈도우**를 써야 해. 데이터 한 번만 훑고 끝내는 1D 이동 평균선 스캐닝이랑 똑같지!

## **🛠️ 네가 해야 할 것 (핵심 구현 목표):**
윈도우 안의 상태를 나타내는 가장 중요한 공식 하나만 기억해.
> **(현재 윈도우 크기) - (윈도우 내 가장 많이 등장한 알파벳 개수) $\le$ 치트키 횟수 $k$**

이 공식이 성립하면 치트키로 전부 같은 글자를 만들 수 있다는 뜻이야!
1.  `right` 포인터가 오른쪽으로 전진하면서 윈도우를 늘린다.
2.  새로 들어온 알파벳을 딕셔너리에 카운트하고, `max_count`를 갱신한다.
3.  **한도 초과 검사:** 윈도우 크기에서 `max_count`를 뺀 값이 $k$보다 크면? 치트키로도 커버가 안 된다는 뜻!
4.  **수축:** `left` 포인터가 가리키던 알파벳 카운트를 1 빼주고, `left`를 오른쪽으로 당겨서 윈도우를 좁힌다.

## **🧰 필요한 파이썬 내장 함수/문법 (범용 사용법):**
**`collections.defaultdict(int)`**

파이썬 기본 딕셔너리는 없는 키(Key)를 조회하거나 더하려고 하면 `KeyError`를 뱉고 뻗어버려. 하지만 `defaultdict(int)`를 쓰면, 처음 보는 키라도 자동으로 기본값 `0`을 세팅해 줘서 조건문 없이 바로 덧셈이 가능해!

```python
import collections

# 일반 딕셔너리였다면 if c in dic: dic[c] += 1 처럼 썼어야 함.
counts = collections.defaultdict(int)
counts['A'] += 1  # 'A'가 없었지만 자동으로 0 -> 1이 됨!
print(counts['A']) # 출력: 1
```

# 3단계: 실행 가능한 코드 스켈레톤 제공 (Runnable Skeleton)

자, `solution.py`에 아래 뼈대 복붙하고 딱 3개의 빈칸을 채워봐! 시계열 윈도우가 고무줄처럼 늘어났다 줄어들었다 하는 걸 상상하면서 짜는 거야.

```python
import collections

class Solution:
    def characterReplacement(self, s: str, k: int) -> int:
        left = 0
        counts = collections.defaultdict(int)
        max_count = 0 
        max_len = 0   
        
        # right 포인터가 0부터 끝까지 한 칸씩 전진 (윈도우 확장)
        for right in range(len(s)):
            # 1. 새로 윈도우에 들어온 알파벳(s[right])의 개수를 1 증가시킨다.
            # ...
            
            # 2. 현재 윈도우 내 단일 알파벳 최대 빈도수(max_count) 갱신
            # 힌트: 기존 max_count와 방금 추가된 알파벳의 개수 중 큰 값을 선택
            # ...
            
            # 3. 🚨 치트키 한도 초과 검사! 
            # (현재 윈도우 크기) - (가장 많이 나온 알파벳 개수) > k 라면?
            if (right - left + 1) - max_count > k:
                # 3-1. 윈도우에서 빠져나갈 맨 왼쪽 알파벳(s[left])의 개수를 1 뺀다.
                # ...
                
                # 3-2. left 포인터를 한 칸 오른쪽으로 당겨서 윈도우를 줄인다.
                # ...
                
            # 4. 안전한 윈도우가 되었으므로, 최대 길이(max_len)를 갱신
            max_len = max(max_len, right - left + 1)
            
        return max_len

if __name__ == "__main__":
    sol = Solution()
    
    # Test Case 1
    print("TC 1:", sol.characterReplacement("ABAB", 2)) # 예상 결과: 4
    
    # Test Case 2
    print("TC 2:", sol.characterReplacement("AABABBA", 1)) # 예상 결과: 4
```

# 📝 [복습 노트] 33. 가장 긴 반복 문자 대체 (LeetCode 424)
**날짜:** 2026-04-08
**소요 시간:** 10분 컷 
**난이도:** Medium 🚨

## 1. 🎯 핵심 로직: 슬라이딩 윈도우 (고무줄 당기기)
* **문제점:** 무지성으로 부분 문자열을 다 확인하면 $O(N^2)$으로 타임아웃 발생.
* **해결책:** `left`, `right` 두 개의 포인터를 사용해 데이터 전체를 한 번만 훑는 **$O(N)$ 슬라이딩 윈도우**를 구축한다.

## 2. 🚨 슬라이딩 윈도우 절대 공식
* **유효성 검사:** `(윈도우 크기) - (가장 많이 등장한 알파벳 수) <= k`
* **확장과 수축:**
    * 조건 만족: `right`를 우측으로 밀며 윈도우 확장!
    * 조건 불만족: `counts[s[left]]`를 1 빼고, `left`를 우측으로 당겨 윈도우 수축!

## 3. 🤯 `max_count` 갱신의 비밀 (신기록 유지)
* `left`가 당겨지더라도 `max_count`는 굳이 감소시키지 않는다. 
* 이유: 우리의 목표는 '가장 큰 윈도우 길이'를 찾는 것이다. `max_count`가 기존 기록을 경신하지 못한다면, 윈도우 크기도 커질 수 없으므로 과거의 최대값을 유지해도 결과에 영향을 미치지 않는다. (극강의 최적화 포인트!)

## 4. 📊 Big-O (시공간 복잡도)
* **시간 복잡도 $O(N)$:** `left`와 `right` 포인터가 문자열을 최대 한 번씩만 순회함.
* **공간 복잡도 $O(1)$:** `counts` 딕셔너리에는 최대 26개의 대문자 알파벳만 저장되므로, $O(26) = O(1)$로 처리됨.
