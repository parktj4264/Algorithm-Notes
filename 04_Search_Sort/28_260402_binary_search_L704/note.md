# 2단계: 문제 및 가이드 제시 (The 'Up/Down Game' Perspective)

### 🚀 문제 704: Binary Search (이진 검색)
### 🔗 **링크:** [https://leetcode.com/problems/binary-search/](https://leetcode.com/problems/binary-search/)

## **📜 문제 설명 (Mission Briefing):**
오름차순으로 정렬된 정수 배열 `nums`와 찾고자 하는 목표값 `target`이 주어져. 
배열 안에 `target`이 존재하면 **그 인덱스를 반환**하고, 없으면 **`-1`을 반환**하는 문제야.
단, 시간 복잡도는 반드시 **$O(\log N)$**으로 짜야 해! (그냥 `for`문으로 하나씩 훑는 $O(N)$은 탈락!)

## **🎯 오늘 부술 로직 (양 끝에서 조여오기):**
이건 술자리에서 하는 '업/다운(Up/Down) 게임'이랑 완전히 똑같아. 
1부터 100까지 중에 75를 맞춘다고 치자. 바보가 아닌 이상 1부터 부르지 않지? 무조건 절반인 50부터 부르잖아!
50 불렀는데 "UP!" 하면? 1~50은 다 버리고, 51~100 사이의 절반인 75를 부르는 거. 이게 끝이야!

이걸 코드로 구현하려면 **투 포인터(`left`, `right`)**가 필요해.
1. `left`는 배열의 맨 앞, `right`는 배열의 맨 끝 인덱스를 가리킨다.
2. 두 포인터의 정중앙인 `mid`를 구한다.
3. `nums[mid]`가 `target`보다 작으면? ➔ **"UP!"** ➔ `left`를 `mid + 1`로 땡겨온다!
4. `nums[mid]`가 `target`보다 크면? ➔ **"DOWN!"** ➔ `right`를 `mid - 1`로 땡겨온다!
5. `nums[mid] == target`이면 정답!

# 3단계: 실행 가능한 코드 스켈레톤 제공 (Runnable Skeleton)

형, 솔직히 이 문제는 개념 자체는 엄청 쉬워. 근데 왜 굳이 이걸 지금 푸냐면, **"이진 검색 무한 루프 방지 절대 템플릿"**을 손에 완벽하게 익히기 위해서야. 여기서 부등호 하나 삐끗하면 나중에 Medium 문제 가서 무한 루프 지옥에 빠지거든!

```python
from typing import List

class Solution:
    def search(self, nums: List[int], target: int) -> int:
        # 양 끝점(인덱스) 세팅
        left = 0
        right = len(nums) - 1
        
        # 🚨 [국룰 템플릿] left와 right가 엇갈리기 전까지 반복! (등호 <= 가 핵심!)
        while left <= right:
            # 1. 정중앙 인덱스 구하기 (소수점 버림)
            mid = (left + right) // 2
            
            # 2. 빙고! 찾았으면 인덱스 반환
            if nums[mid] == target:
                return mid
                
            # 3. UP! (내가 본 숫자가 타겟보다 작으면, 탐색 범위를 오른쪽 절반으로!)
            elif nums[mid] < target:
                pass # left 포인터를 어디로 옮겨야 할까?
                
            # 4. DOWN! (내가 본 숫자가 타겟보다 크면, 탐색 범위를 왼쪽 절반으로!)
            else:
                pass # right 포인터를 어디로 옮겨야 할까?
                
        # 엇갈릴 때까지 다 뒤졌는데도 못 찾았다면?
        return -1


if __name__ == "__main__":
    sol = Solution()
    
    # Test Case 1: 타겟이 있는 경우
    nums1 = [-1, 0, 3, 5, 9, 12]
    target1 = 9
    print("TC 1:", sol.search(nums1, target1)) # 예상 결과: 4 (9의 인덱스)
    
    # Test Case 2: 타겟이 없는 경우
    nums2 = [-1, 0, 3, 5, 9, 12]
    target2 = 2
    print("TC 2:", sol.search(nums2, target2)) # 예상 결과: -1
```

빈칸에 `left`랑 `right` 포인터 땡겨오는 것만 쇽쇽 채워 넣어봐! 
주의할 점: 그냥 `mid`로 옮기면 안 되고, `mid`는 이미 검사했으니까 **`mid + 1`** 또는 **`mid - 1`**로 확실하게 한 칸 건너뛰어서 옮겨야 해!

# 📝 [복습 노트] 28. 이진 검색 (LeetCode 704)
**날짜:** 2026-04-02
**소요 시간:** 5분 컷 (풋살 직후 아드레날린 버프 ⚽🔥)

## 1. 🎯 핵심 로직: 투 포인터 Up/Down 게임
* **문제점:** 정렬된 배열에서 $O(N)$으로 처음부터 끝까지 다 찾는 건 하수.
* **해결책:** 시작점(`left`)과 끝점(`right`)을 잡고, 정중앙(`mid`)을 찔러본 뒤 절반씩 범위를 날려버리는 **$O(\log N)$** 탐색을 한다.

## 2. 🚨 이진 검색 '절대 템플릿' (무조건 외울 것!)
1. **반복 조건:** `while left <= right:` (등호 `=`가 빠지면 숫자 1개 남았을 때 검사를 안 하고 종료됨!)
2. **중앙값:** `mid = (left + right) // 2` (파이썬은 오버플로우 걱정이 없어서 맘 편히 더하고 나누면 됨)
3. **범위 축소:** * 내가 본 숫자보다 타겟이 크면? ➔ `left = mid + 1` (오른쪽 탐색)
   * 내가 본 숫자보다 타겟이 작으면? ➔ `right = mid - 1` (왼쪽 탐색)
   * **주의:** `mid`는 이미 검사했으므로 반드시 `+1`, `-1`을 해줘야 무한 루프에 빠지지 않는다!

## 3. 📊 Big-O (시공간 복잡도)
* **시간 복잡도 $O(\log N)$:** 매번 탐색 범위가 절반으로 줄어듦 (100만 개 데이터도 20번이면 탐색 끝!)
* **공간 복잡도 $O(1)$:** 추가 배열 없이 포인터(인덱스 변수)만 사용.
