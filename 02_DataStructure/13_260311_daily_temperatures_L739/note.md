### 2단계: 문제 및 가이드 제시 (The 'For-loop' Perspective)

### 🚀 문제 13: 일일 온도 (Daily Temperatures)

### 🔗 **링크:** [https://leetcode.com/problems/daily-temperatures/](https://leetcode.com/problems/daily-temperatures/)

## **📜 문제 설명 (Mission Briefing):**

매일의 온도가 담긴 리스트 `temperatures`가 주어져. 네가 해야 할 일은 **"더 따뜻한 날씨를 맞이하려면 며칠을 더 기다려야 하는가?"**를 배열로 반환하는 거야. 만약 미래에 더 따뜻한 날이 없다면, 그 자리는 0으로 둬야 해.

* **예시 1:** `temperatures = [73, 74, 75, 71, 69, 72, 76, 73]`
* 73도(0일차) -> 다음 날 74도니까 **1**일 대기.
* 74도(1일차) -> 다음 날 75도니까 **1**일 대기.
* 75도(2일차) -> 뒤로 71, 69, 72 지나서 76도(6일차)가 되어야 따뜻해지니까 **4**일 대기.
* 결과: `[1, 1, 4, 2, 1, 1, 0, 0]`



## **🎯 오늘 부술 for문 (Big-O 관점):**

문제 제약 조건을 보면 온도의 개수(배열 길이)가 최대 $10^5$개야.
아무 생각 없이 "오늘 온도 잡고, 내일 모레 글피 쭉~ 뒤지면서 더 큰 거 나오면 멈춰야지!" 하고 이중 for문을 돌리면 시간 복잡도는 $O(N^2)$이 돼. $10^5 \times 10^5 = 10^{10}$번의 연산? 리트코드 서버가 바로 'Time Limit Exceeded(시간 초과)' 뱉고 컷해버려.

**어떻게 $O(N)$으로 찢을 것인가?**
메모리(공간 복잡도 $O(N)$)를 제물로 바쳐서 시간(속도)을 연성해야 해. **Stack**을 '아직 더 따뜻한 날을 찾지 못해 대기 중인 날들의 인덱스 저장소(대기실)'로 써봐. 배열을 단 한 번만 순회($O(N)$)하면서, 현재 날씨가 Stack 대기실에 있는 날씨보다 따뜻하면? 대기실 인원들을 팝(pop) 시켜주면서 날짜 차이(현재 인덱스 - 대기실 인덱스)를 정답 배열에 적어주는 거지. 각 날짜는 Stack에 한 번 들어가고 한 번 나오니까 완벽한 선형 시간 $O(N)$이 보장돼.

## **🛠️ 네가 해야 할 것:**

1. 정답을 담을 배열 `answer`를 `temperatures` 길이만큼 0으로 초기화해 둬. (이러면 못 찾는 경우 알아서 0이 됨)
2. 인덱스를 저장할 빈 리스트(Stack)를 하나 만들어.
3. `temperatures`를 순회하면서, **"Stack에 요소가 있고 && 현재 온도가 Stack 맨 위 인덱스의 온도보다 높을 때"**까지 `while`문으로 Stack을 뽑아내(pop).
4. 뽑아낸 인덱스 위치의 `answer` 값에 `현재 인덱스 - 뽑아낸 인덱스`를 기록해.
5. 현재 인덱스를 다시 Stack에 넣어(push).

## **🧰 필요한 파이썬 내장 함수/문법 (범용 사용법):**

R에서 `seq_along`이나 데이터프레임 인덱스 다루던 것처럼, 파이썬에서는 인덱스와 값을 동시에 뽑을 때 `enumerate()`를 밥 먹듯이 써야 해. Stack의 마지막 요소(맨 위)를 확인할 때는 `stack[-1]`을 쓰면 돼.

```python
# enumerate()와 Stack 기본 조작법 예시
my_list = [10, 20, 30]
stack = []

for i, value in enumerate(my_list):
    print(f"인덱스: {i}, 값: {value}")
    stack.append(i)  # 파이썬 list의 append가 스택의 push 역할
    
# 스택 맨 위 요소 확인 (제거하지 않음)
top_element = stack[-1] 

# 스택 맨 위 요소 제거 및 반환
popped_element = stack.pop() 

```

### 💡 코드 리뷰 & Big-O 팩트 체크

코드 로직은 **100점 만점에 100점**이야. 정확히 의도한 대로 짰어.
다만 딱 하나, Pythonic한 관점에서 `while( stack and (tem > temperatures[stack[-1]]) ):` 이렇게 C언어나 R처럼 조건식에 괄호를 치는 건 파이썬에서는 약간 '아재 냄새'가 나. 파이썬은 `while stack and tem > temperatures[stack[-1]]:` 처럼 괄호 없이 깔끔하게 쓰는 걸 선호해.

**가장 중요한 시간 복잡도($O$) 분석을 해보자.**
네 코드를 보면 `for`문 안에 `while`문이 있지? 여기서 알고리즘 뉴비들은 보통 *"어? for문 안에 while이 도니까 이거 $O(N^2)$ 아니야?"* 라고 착각해.

근데 잘 생각해 봐. 어떤 인덱스 `i`든 간에 Stack에 **들어가는 횟수(append)는 딱 한 번**이야. 그리고 Stack에서 **빠져나오는 횟수(pop)도 딱 한 번**이지.
즉, `temperatures`의 길이가 $N$일 때, 아무리 `while`문이 뺑뺑이를 돌아도 전체 과정에서 `push`와 `pop`은 각각 최대 $N$번만 일어나. 총 연산 횟수는 많아 봐야 $2N$이라는 소리야.

따라서 이 코드의 시간 복잡도는 상수를 버리고 완벽한 **$O(N)$**이 돼. $10^5$개의 데이터라도 고작 20만 번 연산으로 0.01초 컷을 내버린 거지. 무지성 이중 for문($O(N^2)$, 100억 번 연산)을 메모리(Stack, $O(N)$ 공간 복잡도)를 제물로 바쳐서 시간 복잡도를 찢어버린 완벽한 예시야.

### 🧠 공간통계/시계열 데이터로 이해하는 단조 스택

네 주석에 `"# 계속 덮어쓰기 의도 (..?)"`라고 적어놨지? 그게 정확해.

통계학에서 확률과정(Stochastic Process) 다룰 때, 특정 상태에 도달할 때까지 걸리는 시간을 **'최초 도달 시간(First Hitting Time)'**이라고 하잖아? 단조 스택이 정확히 그 역할을 해.

1. **Stack은 '미해결 사건 대기실'이야.** 아직 더 따뜻한 날(First Hitting Time)을 맞이하지 못한 날짜(인덱스)들이 차곡차곡 쌓여 있어.
2. 현재 온도(`tem`)가 Stack 맨 위 온도보다 높다? 이건 드디어 **임계점(Threshold)을 돌파**했다는 뜻이야.
3. 그러면 `while`문이 돌면서 조건을 만족하는 과거의 미해결 사건들을 한 방에 청산(`pop`)해버리고, 얼마나 걸렸는지(`i - idx`)를 기록하는 거지.

결과적으로 Stack 내부의 온도들은 항상 '내림차순'을 유지하게 돼(그래서 이름이 '단조' 스택이야).
